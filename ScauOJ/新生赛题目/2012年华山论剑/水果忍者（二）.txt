11165 水果忍者（二）
时间限制:1000MS  内存限制:65535K
提交次数:123 通过次数:28 

题型: 编程题   语言: 无限制



Description
    能够看到这题的师弟，想必能们已经过了新生赛之初出茅庐，那么，还记得Ly大神的那道水果忍者么？
先赞美一下Ly大神吧，他是一个非OI的神牛，但是，却比OI的厉害，还记得12年的校赛，Ly大神差点点就冠军了，成为了华农史上第一个在大一就能冲击冠军的神牛，
最重要的是，他的图论，不是一般的牛逼，经常BS图论白痴胖子。他的PK等级在11里面已经达到了巅峰了！
现在，我来给你们讲一个故事：
    从前有个很喜欢玩水果忍者的人，他一有空就拿出手机玩水果忍者，而且，玩得特别厉害，街机模式的最高记录能刷到1200，你说呢？作为他的队友，我有必要去了
解一下这位大神到底是怎么玩出这种神一般的记录的。
    一天，这位大神又在玩水果忍者了，我偷瞄了一下，发现，原来，他有必杀技的！当他切到一个狂热香蕉的时候，必杀就出现了！
我仔细分析，发现，原来，这个必杀技，是用一个矩形的形状来切水果的，据说这样切可以减少屏幕阻力。
啊，在一旁偷看，被大神发现了，大神知道我偷学了他的必杀，他就开始为难我了，说：“等一下，我玩到一半，就按暂停，然后，你来数一下，这一屏幕里面，
你可以放多少个必杀！时间限制1s”。
    啊啊啊，我凌乱了，那密密麻麻的水果，怎么数啊？
    简化一下吧，就是，给你一个屏幕（二维坐标系），屏幕上有很多水果（点），必杀技的那个矩形的四个顶点必须是以屏幕上的水果为顶点，现在要你算一下，
对于这个屏幕，你能放多少种不同的必杀，也就是有多少个不同的矩形（只要任意一点的坐标不同均视为不同矩形）。
嘻嘻，我是不会告诉你这位大神叫作Ly的！

（出题人 胖子）




输入格式
有多组数据，第一行为一个数T（T<=10），表示有多少组数据。
接着，对于每一组数据，有一个数N（1<=N<=1000），表示平面上有多少个点。
接下来有N行，每行两个数x，y(abs(x，y)<=1e4)，表示每一个点的坐标。
输入全部为整数。



输出格式
对于每一组数据，只输出一个数k（k<10^9），占一行，表示这些点能构成多少个不同的矩形。



输入样例
1
6
0 0
0 1
0 2
1 0
1 1
1 2




输出样例
3
